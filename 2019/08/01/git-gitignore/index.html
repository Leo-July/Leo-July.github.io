git .gitignore 设置了无效

8月 01, 2019 发布在 git

当我们将 .gitignore 文件配置好后，却往往不能失效。

原因： .gitignore 只能忽略那些没有被追踪(track)的文件，因为 git 存在本地缓存，如果文件已经纳入了版本管理，那么修改 .gitignore 是不能失效的。
解决办法：那么解决方案就是要将 git 的本地缓存删除，然后重新提交。
